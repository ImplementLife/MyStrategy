Этот файл-документация к проэкту "MyStrategy"
(Моя версия стратегической игры в реальном времини)

На данный момент необходима оптимизация визуальной части,
расширение функционала...

Последнее в разработке была реализация Железной Дороги
(прохождение вагонами поворотов)

По итогу хочется видеть масштабирование игровой области
с переходом на вид Тактической Военной Карты...

Из основного:

- Класс "Updater" -> Занимается обработкой объектов в ключе
зависимости от времини (Необходима дороботка)...
Присутствует примитивная балансировка по времини...

- Класс "Drawer"  -> Занимается графической состовляющей
объектов (Необходима дороботка)...
В часности отдельно можно выделить Класс "DrawerMap"
он отвечает за отрисовку Основной игровой карты,
Карта генерируется с помощью "Шума Перлина"
Класс "draw.Camera" - соответственно названию...

- Класс "lib.math.Vec2D" -> Основные Векторные операции

- Классы "game.objects.GameObject" & "game.objects.Id" -> Основная объектная область
Все Игровые Юниты наследуются от "game.objects.GameObject" так как он занимается
Сортировкой и упорядочиванием объектов


- GUI Игры -> JPanel на котором рисуем игровые процессы и на
который нацеплены слушатели (клава мишь)




//=============================================//


Все юниты сейчас для тестов добывляются в классе "Drawer"
В методе "start(JPanel)" (Это временная мера вместо
нормальной загрузки карты и объектов)





В пакете "game.unit.working.vehicle.train" жутко недоработанная механика
движения вагонов по ЖД путям (Основная связь между
Классами "Wagon", "RailWay", "WayPoint")
